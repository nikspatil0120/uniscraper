OVERVIEW
--------
This system now supports Google Vertex AI as the primary extraction method,
as recommended by the project reviewers (Res). Vertex AI provides:

  ✓ Higher quotas than standard Gemini API
  ✓ Better reliability for enterprise applications
  ✓ More stable performance at scale
  ✓ Dedicated support for production use

ARCHITECTURE
------------
The extraction pipeline now uses a 4-tier fallback system:

  TIER 1: Vertex AI Gemini       (primary - best quotas & reliability)
       ↓  (if not configured or fails)
  TIER 2: Gemini API              (standard cloud API)
       ↓  (if rate limited)
  TIER 3: Groq Llama              (fast cloud, separate quota)
       ↓  (if unavailable)
  TIER 4: Ollama Qwen2.5          (local, no quota limits)

SETUP STEPS
-----------

STEP 1: Get Google Cloud Credentials
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Option A - Company Provides Account (RECOMMENDED):
  → Ask your company/reviewers for:
     • Service account JSON file
     • Google Cloud Project ID
  → They mentioned they will provide this if implementation works

Option B - Create Your Own:
  1. Go to https://console.cloud.google.com
  2. Create a new project (e.g., "uniscraper-ai")
  3. Enable Vertex AI API:
     • Go to: APIs & Services → Library
     • Search: "Vertex AI API"
     • Click Enable
  4. Create Service Account:
     • Go to: IAM & Admin → Service Accounts
     • Click: Create Service Account
     • Name: vertex-ai-service
     • Role: Vertex AI User
  5. Generate Key:
     • Click on the service account
     • Keys tab → Add Key → Create New Key
     • Choose JSON
     • Download the file

STEP 2: Install Dependencies (ALREADY DONE)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  ✓ Dependencies installed:
    - google-cloud-aiplatform>=1.73.0
    - google-auth>=2.36.0

STEP 3: Place Credentials
~~~~~~~~~~~~~~~~~~~~~~~~~~
  1. Put your service account JSON file here:
     uniscraper-backend/credentials/vertex-key.json

  2. The file should look like:
     {
       "type": "service_account",
       "project_id": "your-project-id",
       "private_key_id": "...",
       "private_key": "-----BEGIN PRIVATE KEY-----\n...",
       ...
     }

STEP 4: Update Configuration
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

TROUBLESHOOTING
---------------

Issue: "Credentials file not found"
→ Solution: Check the path in GOOGLE_APPLICATION_CREDENTIALS
           Make sure to use absolute path with backslashes escaped:
           C:\Projects\uniscrape\uniscraper-backend\credentials\vertex-key.json

Issue: "VERTEX_PROJECT_ID not set"
→ Solution: Add VERTEX_PROJECT_ID=your-project-id to .env
           Project ID is in your service account JSON

Issue: "Permission denied" or "403 Forbidden"
→ Solution: Service account needs "Vertex AI User" role
           Go to IAM & Admin → IAM and add the role

Issue: "Vertex AI not available, falling back to Gemini API"
→ This is NORMAL if Vertex is not configured yet
  The system will work fine with the fallback chain

Issue: "API not enabled"
→ Solution: Enable Vertex AI API in Google Cloud Console:
           https://console.cloud.google.com/apis/library/aiplatform.googleapis.com
